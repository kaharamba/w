Defense-in-Depth Security Architecture for CAN-Based Autonomous Vehicle Control Systems

Status: Oral presentation delivered; manuscript in preparation.

Objective & Research Summary

This work evaluates a defense-in-depth security architecture for CAN-based autonomous vehicle control systems. The system spans distributed throttle, brake, and steering control communicating over a 250 kbit/s CAN bus and integrates layered prevention, detection, and response mechanisms. Validation was performed through controlled attack scenarios including message injection, replay, and malicious command ramping on a physical autonomous vehicle platform.

I conceived and executed the attack methodology used to validate the proposed security architecture. This included performing CAN message injection and replay attacks from an external laptop, successfully taking control of steering and throttle subsystems prior to the activation of security mechanisms. I also supported experimental validation during live demonstrations.

Attack Scenarios Demonstrated

  • CAN message injection from a remote laptop to override steering and throttle commands
  • Replay attacks using spoofed CAN identifiers
  • Command ramping to induce unsafe control behavior

Tools & Systems Used

  • CAN bus (250 kbit/s)
  • Teensy 4.1 microcontrollers
  • Laptop-based CAN injection tooling
  • Distributed embedded control network
  • Autonomous vehicle platform
